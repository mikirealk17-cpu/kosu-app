# 工数管理アプリ

設備組付け作業者向けの、シンプルな工数管理Webアプリです。

## 主な機能

- 工数入力
- 製番から設備名を自動表示
- 開始時間・終了時間・休憩時間から実働時間を自動計算
- 作業内容の選択
- 作業者の選択
- 作業会社の選択
- 工数データの保存
- 入力履歴の編集
- 入力履歴の削除
- CSV出力
- 製番別・設備別・日別・作業者別の集計
- 作業者の追加・編集・削除
- 作業会社の追加・編集・削除
- 作業内容マスタの追加・編集・削除
- 製番マスタの追加・編集・削除
- スマホ向けの入力しやすいUI
- 集計画面の概要カード表示
- スマホで押しやすい入力欄と固定保存ボタン

## 画面説明

### デザイン案確認画面

`design.html`

UIデザイン案の画像を確認するためのページです。
スマホでは画像を横にスクロールして確認できます。

### 工数入力画面

`index.html`

作業者、製番、作業内容、開始時間、終了時間、休憩時間を入力して工数を保存します。
作業会社DBが設定済みの場合は、作業会社も選択して工数ごとに保存します。
作業者を選ぶと、その作業者で前回使った作業会社が自動で選ばれます。
開始時間と終了時間は数字で入力できます。
入力速度を優先し、作業者と日付、製番、時間の順に入力しやすい画面にしています。

例:

- `8` は `08:00` として扱います
- `830` は `08:30` として扱います
- `17` は `17:00` として扱います
- `1730` は `17:30` として扱います

休憩時間は空欄なら0分として扱います。
スマホでは保存ボタンが画面下に表示され、保存後は短い完了メッセージが表示されます。

### 集計画面

`summary.html`

期間を指定して、製番別、製番明細、設備別、日別、月別、作業者日別、作業者別に工数を確認できます。
明細CSV、集計CSV、作業者別設備CSVを選んで出力できます。
作業者別設備CSVでは、期間内に各作業者が関わった設備番号、製番、設備名、実働時間をまとめて確認できます。
総実働時間、入力件数、平均実働、作業者数をカードで確認できます。

### 入力履歴編集画面

`logs.html`

保存済みの工数を一覧表示し、内容を編集・削除できます。
作業者、作業内容、製番で絞り込みできます。
更新・削除後は、対象の日付、作業者、製番、時間、工数が表示されます。
工数入力画面と同じ見た目に揃え、スマホでも編集しやすい構成にしています。

### 管理画面

`admin.html`

作業者、作業会社、作業内容、製番の管理画面へ移動できます。
現在の作業者、作業会社、作業内容、製番、工数記録の登録件数も確認できます。

### 作業者管理画面

`workers.html`

作業者を追加・編集・非表示化できます。
非表示にした作業者は復活できます。
過去の工数データを残すため、作業者は完全削除ではなく非表示で扱います。

### 作業会社管理画面

`billing-companies.html`

請求集計で使う作業会社を追加・編集・非表示化できます。
作業会社は作業者に固定せず、工数入力ごとに保存します。

### 作業内容管理画面

`work-types.html`

作業内容を追加・編集・非表示化できます。
並び順も管理できます。

### 製番管理画面

`seibans.html`

製番と設備名を追加・編集・削除できます。
過去の工数で使われている製番は、削除できない場合があります。

## 技術構成

- HTML

- JavaScript

## ローカルでの起動

```bash
npx serve .
```

表示されたURLをブラウザで開きます。

## Supabase設定

作業者対応には、Supabase側に `worker_master` テーブルと `work_logs.worker_id` 列が必要です。
作業会社対応には、`billing_company_master` テーブルと `work_logs.billing_company_id` 列が必要です。
設定SQLは `SUPABASE_SETUP.sql` にまとめています。
作業会社対応だけを追加したい場合は、`SUPABASE_BILLING_COMPANY_SETUP.sql` を実行してください。
SQL実行後に画面を再読み込みすると、工数入力画面の作業者プルダウンが有効になります。
作業会社テーブルも作成済みの場合は、`billing-companies.html` で作業会社を登録し、工数入力時に作業会社を選べます。
作業会社は作業者マスタに固定せず、工数1件ごとに `work_logs.billing_company_id` として保存します。

## セキュリティメモ

このアプリはログインなしでブラウザからSupabaseへ接続します。
`supabaseClient.js` のキーは公開用キーなので、ブラウザから見えてもよい前提です。

ただし、SupabaseのRLSを有効にし、公開用キーで許可する操作を必要最小限にしてください。
`service_role` などの秘密キーは、絶対にJavaScriptへ書かないでください。
