# 工数管理アプリ

設備組付け作業者向けの、シンプルな工数管理Webアプリです。

## 主な機能

- 工数入力
- 製番から設備名を自動表示
- 開始時間・終了時間・休憩時間から実働時間を自動計算
- 作業内容の選択
- 作業者の選択
- 工数データの保存
- 入力履歴の編集
- 管理者による入力履歴の削除
- Excel出力
- 製番別・製番明細・作業者別の集計
- Supabase Authによるログイン
- 作業者・管理者の権限分け
- 作業者の追加・編集・削除
- 作業内容マスタの追加・編集・削除
- 製番マスタの追加・編集・削除
- スマホ向けの入力しやすいUI
- 集計画面の概要カード表示
- スマホで押しやすい入力欄

## 画面説明

### デザイン案確認画面

`design.html`

UIデザイン案の画像を確認するためのページです。
スマホでは画像を横にスクロールして確認できます。

### 工数入力画面

`index.html`

作業者、製番、作業内容、開始時間、終了時間、休憩時間を入力して工数を保存します。
元請け・単価区分・請求機能のコードとDBは残していますが、工数管理版としての商品化第一段階では通常入力、通常管理画面、集計タブ、単価管理画面、請求確認CSVから外しています。
ログイン後、作業者権限では作業者が本人に固定され、管理画面や集計画面への通常リンクは表示されません。
開始時間と終了時間は数字で入力できます。
入力速度を優先し、作業者と日付、製番、時間の順に入力しやすい画面にしています。

例:

- `8` は `08:00` として扱います
- `830` は `08:30` として扱います
- `17` は `17:00` として扱います
- `1730` は `17:30` として扱います

休憩時間は空欄なら0分として扱います。
保存後は、直近で保存した日付、作業者、製番、設備名、時間、実働時間を画面下側の確認カードに表示します。

### 集計画面

`summary.html`

管理者専用画面です。
期間を指定して、製番別、製番明細、作業者別に工数を確認できます。
出力は、列幅付きの明細Excel、表示中集計Excelだけを通常導線にしています。
請求確認CSVは、金額・請求機能を第2段階として扱うため通常導線から外しています。
総実働時間、入力件数、平均実働、作業者数をカードで確認できます。

### 入力履歴編集画面

`logs.html`

保存済みの工数を一覧表示し、内容を編集・削除できます。
作業者、作業内容、製番で絞り込みできます。
更新・削除後は、対象の日付、作業者、製番、時間、工数が表示されます。
工数入力画面と同じ見た目に揃え、スマホでも編集しやすい構成にしています。
作業者権限では本人の履歴だけ表示し、削除はできません。
管理者権限では全員分の履歴確認、編集、削除ができます。

### 管理画面

`admin.html`

管理者専用画面です。
作業者、作業内容、製番の管理画面へ移動できます。
現在の作業者、作業内容、製番、工数記録の登録件数も確認できます。

### 作業者管理画面

`workers.html`

作業者を追加・編集・非表示化できます。
非表示にした作業者は復活できます。
過去の工数データを残すため、作業者は完全削除ではなく非表示で扱います。

### 元請け管理画面

`billing-companies.html`

第2段階向けの画面です。
仕事を紹介・依頼してきた会社を追加・編集・非表示化できます。
工数管理版としての商品化第一段階では、管理画面の通常導線から外しています。

### 単価管理画面

`rates.html`

第2段階向けの画面です。
時間単価、固定単価（1入力ごと）、請負単価（製番ごと）を登録できます。
同じ条件の使用中単価はSupabase側の制約でも重複登録できないようにします。
工数管理版としての商品化第一段階では、管理画面の通常導線から外しています。

### 作業内容管理画面

`work-types.html`

作業内容を追加・編集・非表示化できます。
並び順も管理できます。

### 製番管理画面

`seibans.html`

製番と設備名を追加・編集・削除できます。
過去の工数で使われている製番は、削除できない場合があります。

## ローカルでの起動

```bash
npx serve .
```

表示されたURLをブラウザで開きます。

## Supabase設定

作業者対応には、Supabase側に `worker_master` テーブルと `work_logs.worker_id` 列が必要です。
元請け対応には、`billing_company_master` テーブルと `work_logs.billing_company_id` 列が必要です。
設定SQLは `SUPABASE_SETUP.sql` にまとめています。
元請け対応だけを追加したい場合は、`SUPABASE_BILLING_COMPANY_SETUP.sql` を実行してください。
第2段階で単価自動適用を追加する場合は、`SUPABASE_RATE_SETUP.sql` を実行してください。
SQL実行後に画面を再読み込みすると、工数入力画面の作業者プルダウンが有効になります。
単価DB関連のコードとSQLは残していますが、金額・請求を安全に扱うまでは通常導線から外しています。

## ログイン・権限管理

第1段階では、権限を `作業者` と `管理者` に分けています。
ログイン画面は `login.html` です。
未ログインで通常画面へ入るとログイン画面へ移動します。

Supabase側の準備は次の順番で行います。

1. `SUPABASE_AUTH_PERMISSION_SETUP.sql` を実行して、`company_master`、`user_profiles`、`company_id` 列を追加する
2. Supabase Authでログインユーザーを作成する
3. `user_profiles` に `auth_user_id`、`role`、作業者の場合は `worker_id` を設定する。第1段階では `company_id` は空でもよい
4. テストユーザーでログインできることを確認する
5. `SUPABASE_AUTH_RLS_POLICIES.sql` を実行して、DB側の閲覧・更新範囲を制限する

## 単価自動適用の設定

単価自動適用の設定順は `BILLING_RATE_SETUP_GUIDE.md` にまとめています。
動作確認用のテストパターンは `BILLING_RATE_TEST_SCENARIOS.md` にまとめています。

## ログイン・権限管理の設計メモ

管理者・会社管理者・作業者で見えるデータを分けるための将来設計は `AUTH_PERMISSION_DESIGN.md` にまとめています。
金額を扱うため、画面で隠すだけでなくSupabase RLSでも閲覧範囲を制限する方針です。
DBに追加する会社マスタ、ユーザープロフィール、会社ID列のSQL案は `SUPABASE_AUTH_PERMISSION_SETUP.sql` に分けています。
RLS本体は `SUPABASE_AUTH_RLS_POLICIES.sql` に分け、ログイン確認後に段階実行します。

## 工数管理版の公開前セキュリティ

工数管理版として商品化する前に、Supabase SQL Editorで `SUPABASE_WORK_TRACKING_RELEASE_SECURITY.sql` を実行してください。
このSQLは、単価・請求関連のテーブルや列を削除せず、匿名ユーザーから `rate_master` と `work_logs` の金額系列を読めない・書けない状態にします。
実行後は `RELEASE_CHECKLIST.md` に沿って、工数入力、履歴編集、集計、明細Excel、表示中集計Excelが壊れていないことを確認します。
ログイン・権限管理を有効にする場合は、続けて `SUPABASE_AUTH_PERMISSION_SETUP.sql`、テストユーザー設定、`SUPABASE_AUTH_RLS_POLICIES.sql` の順に進めます。

## セキュリティメモ

このアプリはブラウザからSupabaseへ接続します。
`supabaseClient.js` のキーは公開用キーなので、ブラウザから見えてもよい前提です。

ただし、SupabaseのRLSを有効にし、公開用キーで許可する操作を必要最小限にしてください。
`service_role` などの秘密キーは、絶対にJavaScriptへ書かないでください。
