# 単価自動適用 設定手順

この手順は、請求金額をCSVに出すための設定順です。
既存の工数データは自動更新しません。

## 1. Supabase SQLを実行する

Supabase SQL Editorで、次の順番で実行します。

1. 作業者・元請け対応が未実行の場合は `SUPABASE_SETUP.sql`
2. 元請け対応だけ未実行の場合は `SUPABASE_BILLING_COMPANY_SETUP.sql`
3. 単価自動適用を追加するために `SUPABASE_RATE_SETUP.sql`

`SUPABASE_RATE_SETUP.sql` は次を追加します。

- `root_company_master`: 大元請けマスタ
- `billing_company_master.root_company_id`: 元請けと大元請けの紐づけ
- `rate_master`: 単価マスタ
- `work_logs` の単価スナップショット列

## 2. 大元請けを登録する

画面: `root-companies.html`

大元請けは、仕事を取ってきた会社です。

例:

- A工業
- D工業

## 3. 元請けを登録する

画面: `billing-companies.html`

元請けは、自分たちや作業者に仕事を紹介・依頼してきた会社です。
登録時に、どの大元請けに紐づく元請けかを選びます。

例:

| 大元請け | 元請け |
| --- | --- |
| A工業 | B設備 |
| A工業 | C設備 |
| D工業 | B設備 |

## 4. 単価を登録する

画面: `rates.html`

単価区分は3種類です。

| 単価区分 | 判定条件 | 金額計算 |
| --- | --- | --- |
| 時間単価 | 大元請け + 元請け + 作業者 | 実働時間 x 単価 |
| 固定単価（1入力ごと） | 大元請け + 元請け + 作業者 | 1入力 x 登録金額 |
| 請負単価（製番ごと） | 大元請け + 元請け + 製番 | CSV内で製番ごとに1回だけ登録金額 |

同じ条件の使用中単価は、DB側でも重複登録できないようにしています。

## 5. 工数を入力する

画面: `index.html`

入力時に次を選びます。

- 作業者
- 大元請け
- 元請け
- 単価区分
- 製番
- 作業内容
- 時間

保存時に単価マスタを確認します。

- 単価0件: 保存しない
- 単価1件: 保存する
- 単価2件以上: 保存しない

保存できた場合は、その時点の単価と金額を工数データに保存します。
後で単価マスタを変更しても、過去の工数金額は勝手に変わりません。

## 6. 請求確認CSVを出力する

画面: `summary.html`

CSV種類で `請求確認CSV` を選びます。

CSVには次が出ます。

- 大元請け
- 元請け
- 単価区分
- 作業者
- 製番
- 設備名
- 作業内容
- 実働時間
- 単価
- 金額
- 請負計上フラグ

請負単価は、同じ大元請け・元請け・製番の組み合わせで1回だけ金額を出します。
2行目以降は `内訳のみ` になり、金額は0円になります。

## 7. 最低限の確認

次の3パターンを確認します。

| 確認内容 | 期待結果 |
| --- | --- |
| 時間単価3000円で2時間入力 | 金額6000円 |
| 固定単価50000円で1件入力 | 時間に関係なく50000円 |
| 請負単価80000円で同じ製番に複数人入力 | CSVで80000円が1回だけ計上 |

異常系も確認します。

- 単価未登録なら保存できない
- 同じ条件の単価は重複登録できない
- 単価情報がない既存データは金額CSVで異常扱いになる
